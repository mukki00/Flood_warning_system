Introduction

The requirement of the second coursework for the module "Cloud Computing and Internet of Things" is the development and implementation of a prototype system utilising both IoT and Cloud Computing. The prototype proposed and developed is an IoT-Enabled Urban Flood Early Warning and Alert System.

The system uses a simulated ESP32 microcontroller (via Wokwi) equipped with multiple environmental sensors to detect flood-risk conditions in real time. Sensor data is transmitted to a Flask API backend over HTTP and MQTT, persisted locally with Redis, and optionally synchronised to Azure Cosmos DB and Azure Event Hub for scalable cloud analytics. A React dashboard provides live visualisation, history charts, and tri-level risk alerts.

The offline-first architecture ensures uninterrupted local operation even when cloud connectivity is unavailable, with a background retry worker draining the Redis queue once connectivity is restored.

Aims & Objectives

The main aim is to design, implement and demonstrate a resilient flood early warning platform combining IoT sensing, edge processing, and cloud analytics. Objectives:

  • Simulate an ESP32 sensor node (Wokwi) with ultrasonic, DHT22, rain and soil-moisture sensors.
  • Implement tri-level risk classification (LOWMODERATEHIGH) on the device.
  • Develop a Flask REST API for sensor data ingestion via HTTP POST and MQTT subscription.
  • Build an offline-first local store (in-memory + Redis cache) with cloud retry queue.
  • Integrate Azure Cosmos DB for persistent cloud storage and Azure Event Hub for stream analytics.
  • Create a React/Vite dashboard with live sensor cards, time-filtered history charts, alert log, and system status.
  • Provide operational scripts for offline detection and zero-touch Redis startup.

Components Used

  • ESP32 DevKit V1 — simulated via Wokwi
  • HC-SR04 Ultrasonic — water-level distance sensor
  • DHT22 — temperature & humidity

Software Design & Development

The software spans three tiers: the ESP32 firmware, the Flask API backend, and the React dashboard. An offline-first design ensures data is never lost when cloud services are unreachable.

ESP32 Firmware (C++ / Arduino)

  • measureWaterLevel() — triggers HC-SR04, converts pulse to cm distance.
  • readDHT() — reads temperature (°C) and relative humidity (%) from DHT22.
  • readADC(pin) — reads 12-bit ADC value from rain / soil potentiometers.
  • computeRisk() — weighted scoring across all sensors; returns LOW / MODERATE / HIGH.
  • updateIndicators(risk) — drives LED pins and buzzer; updates LCD1602 over I2C.
  • postToFlask(payload) — serialises JSON and HTTP POSTs to Flask API endpoint.
  • publishMQTT(payload) — publishes JSON to flood/node1/data topic via HiveMQ.
  • reconnectMQTT() — ensures MQTT client stays connected with auto-reconnect loop.

Flask REST API (Python)

  • ingest_sensor_data() — validates POST payload (7 required fields), stores locally, triggers cloud sync, auto-generates MODERATE/HIGH alerts.
  • get_latest() — returns most recent reading from in-memory store.
  • get_alerts() — returns alert log; supports ?limit= query param.
  • get_status() — returns system health: online/offline, Redis state, cloud targets, reading counts.
  • start_mqtt_client() — spawns background thread subscribing to HiveMQ broker.
  • sync_reading_to_cloud() — writes to Cosmos DB and Event Hub; returns bool success.
  • start_sync_worker() — background thread that drains Redis retry queue when cloud is reachable.
  • offline_store.cache_reading() — pushes reading to Redis list; used as local buffer.
  • offline_store.enqueue_sync_reading() — adds to Redis retry queue on cloud failure.

React Dashboard (JavaScript / Vite)

  • useSensorData(filterKey) — custom hook; polls /api/* every 5 s; manages latest, history, alerts, status state.
  • SensorCard — displays a single metric (water level, temp, humidity, rain) with colour-coded threshold badges.
  • RiskIndicator — renders tri-level risk badge with animated pulse for HIGH risk.
  • SensorChart — line chart (Recharts) of historical readings with time-window filtering.
  • AlertList — scrollable log of MODERATE/HIGH alerts with timestamps and node IDs.
  • RainStatus — visual rain intensity indicator derived from raw ADC value.
  • TimeFilter — toggles between Live, 1 h, 6 h, 24 h history views.

Azure Cloud Integration

  • azure_cosmos_client.py — upserts readings to FloodWarningDB / SensorReadings container; supports range queries by timestamp.
  • azure_eventhub_client.py — batches and sends sensor events to Event Hub (flood-events) for downstream Stream Analytics.
  • cloud_sync.py — orchestrates dual-target sync; returns composite success status.

Offline Resilience

The system operates fully offline. When cloud services are unreachable:

  • All readings are stored in-memory and cached to Redis.
  • Failed cloud syncs are enqueued to the Redis retry queue (flood:sync:readings).
  • A background sync worker retries every 5 seconds, draining the queue in batches of 50 when connectivity is restored.
  • PowerShell scripts (offline-check.ps1) verify Redis and API health; start-redis.ps1 launches a Docker Redis container automatically.
  • The GET /api/status endpoint reports live sync targets, queue depth, and Redis availability.

Conclusion

The design and implementation of this prototype successfully demonstrates a complete IoT-to-Cloud pipeline for urban flood early warning. The system integrates real-time multi-sensor data from a Wokwi-simulated ESP32, a resilient Flask API with offline-first Redis storage, optional Azure Cosmos DB and Event Hub cloud synchronisation, and an interactive React monitoring dashboard.

The offline-first architecture and background retry mechanism ensure zero data loss even under intermittent cloud connectivity — a critical requirement for emergency systems. The tri-level risk classification with local LED/buzzer actuation and remote dashboard alerts provides an end-to-end flood monitoring solution applicable to real urban deployments.

This project has deepened practical expertise in IoT firmware development, RESTful API design, cloud-integrated data pipelines, and frontend real-time monitoring — all core themes of the CS7080 Cloud Computing and Internet of Things module.
